This project is primarily a "scratch my own itch" project, and thus
contributions are broadly not expected. Use parts if it's useful, you get to
keep both parts if it breaks. (See license.) However, for completeness, and to
set a standard, the following apply.

## Code of Conduct

This project follows the Contributor Covenant version 3.0. See
[CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## Policy on LLMs and Similar Large-Scale Trained Models

**No contributions created using** large language models (LLMs) or other
large-scale trained machine learning systems (sometimes called "generative AI"),
or derived from their output, will be reviewed or accepted. Use of LLMs or
similar systems in any project communication, including code review, is
similarly prohibited.

Their use poses a variety of major issues we choose not to debate at this time,
including serious questions about copyright, licensing, and attribution, in
addition to sustainability, future maintainability, suitability for purpose, and
a range of ethical concerns.

While not the most important reason for the author, the most widely-salient
reason is the following. Use of these tools tends to shift the burden of
ensuring a contribution is suitable from a contributor to reviewers during the
code review process. We consider any such shift of this kind to be an
unsustainable waste of limited project resources. For this reason among others,
we ask that you **not submit contributions created with such tools** nor
recommend their use for solving problems in the project space. Any contributions
noted or suspected after submission to be created with such a tool **will be
rejected** without explanation beyond a reference to this policy.

Thanks to the following projects whose policy on these tools most closely
informed our policy. Their policies contain additional informative justification.

  - [Amaranth HDL CONTRIBUTING.txt](https://github.com/amaranth-lang/amaranth/blob/f1014e1038844deaccd2aea1e38fe6759459026c/CONTRIBUTING.txt)
  - [Servo "Contributing" book](https://book.servo.org/contributing/getting-started.html#ai-contributions)
  - [QEMU Code Provenance policy](https://www.qemu.org/docs/master/devel/code-provenance.html#use-of-ai-generated-content)
  - [PostmarketOS AI Policy](https://docs.postmarketos.org/policies-and-processes/development/ai-policy.html)
